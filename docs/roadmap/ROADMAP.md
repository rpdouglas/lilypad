## Governance

Before opening a new phase: read the phase file, run `/read-state`, confirm persona gate from previous phase is passed.

Before closing a phase: all deliverables checked, persona gate answered, `/ticket-close` run to update DECISIONS.md and SITEMAP.md.

Parking lot for work that doesn't belong to an active phase: [BACKLOG.md](BACKLOG.md)
